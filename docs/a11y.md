# Accessibilité web — état & pratiques

Plugin `eslint-plugin-jsx-a11y` posé sur `/client`. La config dans
[client/package.json](../client/package.json) sépare les règles en deux
groupes selon leur sévérité opérationnelle.

## Règles hard (error)

Bloquent la CI. Le code DOIT être corrigé avant merge.

| Règle                         | Pourquoi                                                           |
| ----------------------------- | ------------------------------------------------------------------ |
| `jsx-a11y/alt-text`           | Image sans alt → lecteur d'écran annonce un nom de fichier inutile |
| `jsx-a11y/anchor-has-content` | Lien vide → annoncé comme "lien" sans contexte                     |
| `jsx-a11y/anchor-is-valid`    | `<a>` sans `href` valide → comportement clavier cassé              |
| `jsx-a11y/aria-props`         | Prop ARIA mal orthographiée → ignorée silencieusement              |
| `jsx-a11y/aria-role`          | `role` invalide → l'élément perd toute sémantique a11y             |

## Règles soft (warn)

Visibles dans `npm run lint` mais ne bloquent pas. À traiter en dette
progressive sur les pages les plus utilisées.

| Règle                                             | État actuel | Approche recommandée                                                                             |
| ------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------ |
| `jsx-a11y/label-has-associated-control`           | ~75         | Associer via `htmlFor`/`id` ou wrapper l'input dans `<label>`. Critique pour les screen readers. |
| `jsx-a11y/click-events-have-key-events`           | ~28         | Si élément vraiment cliquable : ajouter `role="button" tabIndex={0} onKeyDown` (Enter + Espace). |
| `jsx-a11y/no-static-element-interactions`         | ~26         | Idem ci-dessus, ou remplacer le `<div onClick>` par un vrai `<button>` stylé.                    |
| `jsx-a11y/no-noninteractive-element-interactions` | ~5          | Pattern modal/dialog : ajouter `role="dialog"` `aria-modal="true"` + Escape handler.             |
| `jsx-a11y/no-autofocus`                           | ~4          | Justifié sur 2FA / code de vérif. Sinon retirer pour ne pas voler le focus aux SR.               |

Le total `npm run lint` doit afficher **0 errors**. Les warnings sont
suivis comme dette technique.

## Patterns à appliquer

### Label + input

```jsx
{/* Avant — label orphelin */}
<label>Nom patient</label>
<input value={form.nom} onChange={...} />

{/* Après — htmlFor/id explicites */}
<label htmlFor="patient-nom">Nom patient</label>
<input id="patient-nom" value={form.nom} onChange={...} />

{/* Ou — label wrap (auto-associé) */}
<label>
  <span>Nom patient</span>
  <input value={form.nom} onChange={...} />
</label>
```

### Div cliquable

```jsx
{
  /* Avant */
}
<div onClick={handle} className="cursor-pointer">
  …
</div>;

{
  /* Après — vraie sémantique de bouton */
}
<div
  role="button"
  tabIndex={0}
  onClick={handle}
  onKeyDown={(e) => {
    if (e.key === "Enter" || e.key === " ") {
      e.preventDefault();
      handle();
    }
  }}
  aria-label="Action explicite"
  className="cursor-pointer"
>
  …
</div>;

{
  /* Mieux — utiliser un vrai <button> stylé */
}
<button type="button" onClick={handle} className="…">
  …
</button>;
```

### Modal

```jsx
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-title"
  tabIndex={-1}
  onKeyDown={(e) => {
    if (e.key === "Escape") onClose();
  }}
  className="fixed inset-0 …"
  onClick={onClose}
>
  <div role="document" onClick={(e) => e.stopPropagation()} className="…">
    <h2 id="modal-title">Titre du dialogue</h2>…
  </div>
</div>
```

### Image décorative vs informative

```jsx
{/* Décoratif — invisible aux SR */}
<svg aria-hidden="true">…</svg>
<img src="ornament.svg" alt="" />

{/* Informatif — texte alternatif descriptif */}
<img src="badge-cpam.png" alt="Conventionné CPAM" />
```

## Audit manuel — pages critiques

À tourner régulièrement avec les DevTools navigateur :

1. **Login** — navigation au clavier (Tab/Shift-Tab), Enter pour soumettre,
   focus visible.
2. **Dashboard** — toutes les cartes accessibles au clavier, badges
   compréhensibles par un lecteur d'écran (VoiceOver / NVDA).
3. **NouveauTransport** (`src/pages/NouveauTransport.jsx`) — formulaire
   complet navigable au clavier, erreurs annoncées.
4. **Factures** — modal de création accessible (Escape ferme, focus piégé
   dans le dialogue).

## Lighthouse / WAVE

`Chrome DevTools → Lighthouse → Accessibility` sur les 4 pages ci-dessus
en mode mobile. Objectif court terme : score ≥ 90 partout. Le contraste
Tailwind (texte slate-600 sur fond white) passe AA, mais slate-400 sur
white-100 échoue — éviter cette combinaison sur du contenu informatif.
